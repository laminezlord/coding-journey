SECTION A: JavaScript Fundamentals & History

1. What is JavaScript?
JavaScript is a high-level programming language used to add logic and interactivity to web pages and applications. It is mainly interpreted/just-in-time compiled by JavaScript engines in browsers and runtimes like Node.js.
HTML structures content, CSS styles it, but JavaScript controls behavior like clicks, animations, and data processing.

2. Brief history of JavaScript
JavaScript was created by Brendan Eich in 1995 at Netscape. It was designed to make web pages interactive and allow simple scripting in the browser.

3. Why the name ‚ÄúJavaScript‚Äù?
The name was chosen for marketing reasons when Java was popular. JavaScript is not related to Java in design or usage. The name helped attract attention.

4. Why JavaScript is the ‚Äúlanguage of the web‚Äù
All major browsers support JavaScript natively. Other languages cannot run directly in the browser without being compiled to JavaScript. JavaScript is the only language browsers execute for client-side logic.

5. Why JavaScript became popular
It is easy to start with, runs in every browser, has a massive community, and a rich ecosystem of libraries and frameworks. It can be used for both front-end and back-end development.

SECTION B: JavaScript as a Programming Language

1. JavaScript as a high-level language
High-level languages manage memory automatically, provide abstraction, and are easy to read and write. Low-level languages require manual memory control and are closer to hardware.

2. High-level vs low-level languages
High-level: JavaScript, Python (used for apps, web, automation).
Low-level: C, Assembly (used for operating systems, drivers, embedded systems).

3. Interpreted language
Interpreted languages run through an engine that translates and executes code at runtime. Compiled languages translate code into machine code before execution.

4. Does JavaScript run line by line?
Modern engines like V8 parse JavaScript, optimize it, compile parts to machine code (JIT), then execute it. It is not simply line-by-line anymore.

5. Dynamically typed
JavaScript decides variable types at runtime. A variable can hold different types at different times without explicit type declaration.

SECTION C: JavaScript Engines & Execution

1. JavaScript engine
A JavaScript engine executes JavaScript code by parsing, compiling, and running it.

2. V8 engine
V8 was developed by Google. It is used in Chrome and Node.js. It made JavaScript fast and suitable for large applications and servers.

3. What happens when JavaScript runs in the browser
Code is loaded ‚Üí parsed ‚Üí compiled/optimized ‚Üí executed by the engine ‚Üí interacts with the browser APIs and DOM.

4. Why JavaScript runs in browsers without installation
Browsers come with built-in JavaScript engines, so users don‚Äôt need to install anything extra.

SECTION D: JavaScript in the Browser vs Outside

1. JavaScript in the browser
The browser provides an environment with a JS engine and Web APIs (DOM, events, fetch).

2. No direct file system access (security)
Browsers block file system access to protect users from malicious websites stealing or modifying files.

3. What is Node.js?
Node.js is a runtime that allows JavaScript to run outside the browser, mainly on servers.

4. Why Node.js is needed
Browser JS is limited to web APIs. Node.js provides server features like file system access, networking, and databases.

5. What each can do
Browser only: DOM manipulation, alerts, page events.
Node.js only: read/write files, create servers, access OS resources.
Reason: different environments and security rules.

SECTION E: Variables in JavaScript

1. Variable (analogy)
A variable is a named container for storing data.
Analogy: a labeled box holding items.

2. Why variables are needed
They store values that programs work with, allowing calculations and logic.

3. var vs let vs const

var: function-scoped, can be redeclared and reassigned

let: block-scoped, can be reassigned

const: block-scoped, cannot be reassigned

4. Why var is unsafe
It allows redeclaration and hoisting issues that cause bugs.

5. Why use const by default
It prevents accidental reassignment and makes code safer and clearer.

6. const objects can be modified
The reference cannot change, but the contents of the object can be changed.

SECTION F: Data Types in JavaScript

1. What are data types
They define the kind of data stored (number, text, etc.).

2. Primitive data types
number, string, boolean, undefined, null, symbol, bigint

3. Primitive vs non-primitive
Primitives store values directly.
Objects store references to values.

4. Loosely/dynamically typed
Types are not fixed and can change at runtime.

5. Type determination at runtime
JavaScript checks the type when the code runs, not before.

SECTION G: Type Conversion & Coercion

1. Type conversion (explicit)
Manually converting types, e.g., Number("5").

2. Type coercion (implicit)
JavaScript automatically converts types during operations.

3. Why coercion is dangerous
It can cause unexpected results like "5" + 1 = "51".

4. == vs ===
== allows type coercion.
=== checks value and type.
=== is safer and preferred.

5. Automatic conversion examples
"5" * 2 becomes 10 because JS converts the string to a number.
